AWS Static Portfolio Architecture

Executive Summary

Architected and implemented a production-grade static website on AWS using CloudFront as a secure global edge layer in front of a private S3 origin. Integrated ACM for TLS (us-east-1 requirement), Route 53 for DNS, and GitHub Actions with OIDC federation for secure, keyless CI/CD deployment. The solution prioritizes security, scalability, cost efficiency, and operational simplicity.

Production-grade static website hosted on AWS using secure, scalable cloud patterns.

Problem

Build a secure, globally available personal portfolio website using AWS infrastructure while following production best practices:

  • No public S3 access
  • HTTPS-only traffic
  • Least privilege IAM
  • Automated CI/CD deployment
  • Low operational cost

Key Architecture Decisions

Why CloudFront instead of direct S3 static hosting?

  • Security: S3 remains private; CloudFront OAC prevents direct bucket access.
  • Performance: Global edge caching reduces latency worldwide.
  • HTTPS enforcement: CloudFront redirects HTTP → HTTPS.

Why OIDC instead of AWS access keys?

  • No long-lived secrets stored in GitHub.
  • Short-lived, temporary credentials via IAM role assumption.
  • Role scoped to specific repository and branch.

Why must ACM be in us-east-1?

  • CloudFront requires custom TLS certificates to be provisioned in us-east-1.

Why does case sensitivity matter in S3?

  • S3 object keys are case-sensitive, meaning /Projects/ and /projects/ are different paths.

Why invalidate CloudFront after deploy?

  • Ensures immediate visibility of changes.
  • Using /* is simple for small sites; larger systems would use targeted invalidation or cache-busting strategies.

AWS Services Used

  • S3 – Private static content storage
  • CloudFront – Global CDN and TLS enforcement
  • ACM – Free public TLS certificate
  • Route 53 – DNS with alias records
  • IAM – Least privilege deployment role

Security Controls

  • Private S3 bucket (no public ACLs)
  • Origin Access Control (OAC)
  • HTTPS-only enforcement
  • MFA enabled for root and admin users

CI/CD Strategy

Site deploys automatically on push to the main branch.

  • GitHub Actions assumes IAM role via OIDC
  • Repository syncs to S3
  • CloudFront invalidation triggered post-deploy

Cost Considerations

  • S3 storage: minimal (static assets only)
  • CloudFront: low traffic footprint
  • ACM: free certificate
  • Estimated monthly cost: typically < $5 under light traffic

Tradeoffs

  • Static site only (no backend)
  • Full cache invalidation used for simplicity
  • No WAF currently attached

Future Improvements

  • Add AWS WAF for enhanced protection
  • Add CloudWatch dashboards and alarms
  • Rebuild infrastructure using Terraform
  • Implement path-based cache invalidation
