AWS Static Portfolio Architecture

Production-grade static website hosted on AWS using secure, scalable cloud patterns.

Problem

Build a secure, globally available personal portfolio website using AWS infrastructure while following production best practices:

  • No public S3 access
  • HTTPS-only traffic
  • Least privilege IAM
  • Automated CI/CD deployment
  • Low operational cost

Architecture Overview

User
  ↓
Route 53 (DNS)
  ↓
CloudFront (CDN + TLS via ACM)
  ↓
S3 (Private Bucket via Origin Access Control)

Architecture Diagram

High-level request flow + CI/CD deployment path.

AWS Services Used

  • S3 – Private bucket hosting static files
  • CloudFront – Global CDN with HTTPS enforcement
  • ACM – TLS certificate (us-east-1)
  • Route 53 – DNS + alias records
  • IAM – OIDC-based role for GitHub Actions

Security Decisions

  • S3 bucket is private (no public ACLs)
  • CloudFront uses Origin Access Control (OAC)
  • HTTPS enforced via redirect policy
  • No long-lived AWS access keys
  • GitHub OIDC federation for short-lived credentials
  • MFA enabled for root and admin users

CI/CD Strategy

The site automatically deploys on push to the main branch.

  • GitHub Actions assumes IAM role via OIDC
  • Syncs repository contents to S3
  • Invalidates CloudFront cache (/*)

Cost Considerations

  • S3 storage: minimal (static assets only)
  • CloudFront: low traffic footprint
  • ACM: free public certificate
  • Estimated monthly cost: typically < $5 at low traffic

Tradeoffs

  • Static site only (no dynamic backend)
  • Full cache invalidation used (could optimize later with path-based invalidation)
  • No WAF currently attached (future enhancement)

Future Improvements

  • Add AWS WAF for additional protection
  • Add CloudWatch metrics dashboard
  • Rebuild infrastructure using Terraform
  • Implement path-based cache control headers
